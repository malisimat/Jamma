Per-LoopTake MIDI start-time quantisation

Non-destructive, realtime-adjustable snap-to-grid for recorded MIDI events — preserves note duration, respects loop boundaries, and updates the 3D MIDI viz live.

37 / 37 tests passing JammaLib Debug x64 — clean build Realtime-safe (lock-free, no heap) Non-destructive (original events untouched) JAM-file persistence — out of scope

What ships

Scope

Per-LoopTake quantisation

Each LoopTake carries its own MidiQuantisationSettings, propagated to every MidiLoop it owns. New MIDI loops opened during recording inherit the active take settings immediately.

Grid

Grain-relative fractions

Snap step is grainSamps / divisor. Six choices: 1 1/2 1/4 1/8 1/16 1/32 — disabled passes timing through bit-exact.

Semantics

Nearest snap, paired shift

NoteOn snaps to the nearest multiple; its matching NoteOff rides the same delta so duration is preserved. Shifts past the loop end clamp to loopLength - 1; negative shifts clamp to 0.

UX

Live toggle, live retarget

Fraction adjusts in realtime via a GuiAction payload [enabled, fractionIdx, grainSamps?]. Audio thread observes flips via a single atomic flag with acquire/release ordering.

Visualisation

3D MIDI viz follows the grid

The instance buffer driving MidiModel is rebuilt from the quantised event stream whenever settings change, so the spatial note-spans visibly snap to the new grid.

Non-destructive

Disable restores bit-exact timing

The original recorded events are never overwritten. Toggling quantisation off flips the audio thread back to the untransformed event array.

Design — dual-buffer with atomic publish

The hot path must not allocate, lock, or branch on unsynchronised state. The implementation uses a parallel preallocated event buffer and a single std::atomic<bool> flag to publish it.

Step 1
Record

_events stores raw incoming events. Untouched by quantisation.

Step 2
Settings change (UI thread)

Flip _useQuantised → false (release), rebuild _quantisedEvents in place.

Step 3
Publish

If step > 0 and loop has length, flip _useQuantised → true (release).

Step 4
Play (audio thread)

ReadBlock snapshots the flag once (acquire) and reads from the matching buffer for the whole block.

Why this is safe in callback context

  • Both _events and _quantisedEvents are std::array<MidiEvent, 4096> — zero allocation, fixed cache footprint.
  • The audio thread never writes either buffer. The UI thread is the sole writer; it disables the flag before mutating _quantisedEvents so the audio thread can never see a half-built buffer.
  • EmitEventsInRange takes a single load of _useQuantised per block, so timing is consistent across the entire ReadBlock window.
  • No mutexes, no condition_variable, no allocation — matches the Jamma hotpath rules verbatim.

Snap math

// step = grainSamps / divisor; 0 == disabled / no-op
std::uint32_t QuantiseSampleOffset(std::uint32_t offset,
                                   std::uint32_t step,
                                   std::uint32_t loopLength) noexcept
{
    if (step == 0 || loopLength == 0) return offset;
    const auto half    = step / 2u;
    const auto snapped = ((offset + half) / step) * step;     // round-nearest
    return snapped % loopLength;                              // wrap at boundary
}

Nearest-multiple snap with half-up tie break; wrap-around so notes near the loop end snap forward to the start instead of escaping the window.

// Per-(channel,note) slot remembers the NoteOn shift so the matching
// NoteOff rides the same delta. Preserves recorded duration. No alloc.
std::array<std::int64_t, 16 * 128> pendingDelta{};
std::array<bool,         16 * 128> hasPending{};

if (ev.IsNoteOn()) {
    const auto q = QuantiseSampleOffset(ev.sampleOffset, step, loopLength);
    pendingDelta[slot] = (int64_t)q - (int64_t)ev.sampleOffset;
    hasPending[slot]   = true;
    ev.sampleOffset    = q;
} else if (ev.IsNoteOff() && hasPending[slot]) {
    const auto shifted = (int64_t)ev.sampleOffset + pendingDelta[slot];
    ev.sampleOffset = (shifted < 0)           ? 0u
                    : (shifted >= loopLength) ? loopLength - 1u
                                              : (uint32_t)shifted;
    hasPending[slot] = false;
}
