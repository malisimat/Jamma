What Changed

  • Added VstAudioBuffers.h helpers for mono-flexible and exact bus layouts, channel copying, and output folding.
  • Added VstPlugin::ProcessBlockMulti() and exact multichannel station processing.
  • Changed loop VST loading to use HostedLayoutMode::MonoFlexible so mono loops can drive stereo plugins and fold output back to mono.
  • Added LoopTake-level VST chains that process all take loop channels together before routing to station buses.
  • Persisted LoopTake VST chains in .jam files alongside existing loop and station VST chains.

Crash Avoidance

  • Kept Steinberg::IPtr<IPluginFactory> instead of the raw factory pointer from ead0169.
  • Kept FUNKNOWN_CTOR, FUNKNOWN_DTOR, and IMPLEMENT_FUNKNOWN_METHODS for host callback adapters.
  • Added targeted failed-load cleanup through ResetLoadedObjects() instead of calling full Unload() from partial Load() failure paths.
  • Ported narrow shutdown cleanup: close VST editors before audio teardown, detach GL debug callbacks before releasing the context, and move FreeConsole() out of WM_DESTROY.

LoopTake VST Path

  • LoopTake selection now inserts one VST chain on the take, not one plugin per child loop.
  • The take reads all loop buffers into preallocated scratch channels, applies master gain, processes the exact bus through VstChain::ProcessBlockMulti(), then routes each processed channel through its mixer.
  • A two-loop take can now feed a stereo VST as left/right channels before being routed onward.

Known Edges

  • LoopTake VST loading is currently immediate, matching Loop-level VST loading, rather than job-thread/preinit based like Station VST loading.
  • Runtime VST editor workflows for LoopTake chains still need UI affordances beyond insert and persistence.
  • The console-control smoke-test exit code is separate from the original VST teardown crash and may be harness-specific.
