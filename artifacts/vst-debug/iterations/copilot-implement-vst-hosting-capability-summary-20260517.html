What Changed

  • Added JammaLib/src/vst/VstAudioBuffers.h with hosted layout negotiation helpers and buffer copy/fold helpers.
  • Updated VstPlugin to negotiate VST input/output bus widths separately, call setBusArrangements() before setupProcessing(), read back active arrangements, and process mono, stereo, or exact-match multichannel buffers using preallocated scratch storage.
  • Updated loop VST loading to use HostedLayoutMode::MonoFlexible, allowing mono loop audio to drive stereo-capable plug-ins safely and fold output back to mono.
  • Updated station VST loading and playback to use exact full-bus processing in one pass via ProcessBlockMulti() instead of pairwise mono/stereo slices.
  • Added focused tests for the new layout and buffer helpers and wired them into JammaLib.Tests.

Startup-Safe Adjustments

  • The initial first-fix port reproduced the startup crash path described in the branch summary.
  • To avoid that regression, VstPlugin now treats GetPluginFactory() as a module-owned raw pointer instead of a cached IPtr<IPluginFactory>.
  • Host callback adapters for IPlugFrame and IComponentHandler were made host-owned and non-destroyable with fixed addRef()/release() behavior.
  • Teardown order in VstPlugin::Unload() was tightened just enough to release controller hookups and interfaces before module unload.
  • No Main.cpp, Window.cpp, Scene.cpp, or VstEditorWindow.* changes were brought over, so the later quit-path mitigation work was intentionally not ported.
