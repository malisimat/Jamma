Quantisation Visual Implementation

Implemented for issue #94: per-LoopTake quantisation gates now align with each take instead of rotating as one master-loop overlay, and the overlay is hidden until explicitly needed.

What Changed

  • LoopTake exposes visual loop length, phase, and radius helpers so the quantisation overlay can match the representative take timing.
  • QuantisationModel now flattens per-take gate instances with independent angle, vertical placement, height scale, and radius scale.
  • The quantisation shader now separates frame, backing rect, and central column appearance, with the backing rect rendered more opaque than the frame.
  • Station builds per-LoopTake overlay input on the render thread and no longer uses master-loop phase for all gates.
  • Scene controls overlay alpha globally: Ctrl holds it at full visibility; release fades over 2 seconds; tap tempo and first local quantisation pulse it immediately.

Issue Details Covered

  • Longer LoopTakes receive more gates, derived from loopLength / seedSamps.
  • Each LoopTake uses its own LoopIndexFrac(), so its gates rotate at that take's rate.
  • Gate height is aligned to the LoopTake row instead of spanning the whole station.
  • The central station column is rendered as its own shader-selected visual part.
  • The overlay is no longer permanently visible.
