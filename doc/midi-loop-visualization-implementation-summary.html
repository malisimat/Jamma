What Was Added

MIDI Logic

Note spans from raw events

Added MidiNoteSpan and ExtractMidiNoteSpans to convert fixed-capacity MidiEvent storage into displayable note durations. The logic handles note-on/off pairing, velocity-zero note-offs, loop-end clamping, out-of-window filtering, orphan note-offs, overlap handling, and dense event stability.

Safety

Non-real-time read surface

MidiLoop now exposes TryGetEvent and Revision so visualization code can copy event data outside the hot path without exposing mutable storage.

Rendering

Generic instance attributes

GuiModel now has opt-in per-instance attribute buffers using glVertexAttribDivisor. Existing non-instanced models stay on the old draw path.

3D Model

MidiModel shell

Added a model sibling to LoopModel that builds a reusable arc-prism mesh and converts note spans into per-instance data: start, duration, pitch offset, velocity, radius, thickness, and height.

Shaders

Single shader for scene, picker, highlight

The new midi_note shader places arcs around the Y axis in the vertex shader and uses a procedural velocity gradient in the fragment shader. Picker and highlight modes are switched by RenderMode.

Issues Encountered

  • MSBuild target name: /t:JammaLib.Tests fails because of the dot. /t:JammaLib_Tests works.
  • Worktree build quirk: Avoid explicit trailing SolutionDir; it can upset vcpkg manifest args in this worktree.
  • Private MIDI storage: MidiLoop had no event read surface, so a narrow copy API was added instead of exposing mutable storage.
  • Shared GL path risk: GuiModel changes compile and tests pass, but manual app rendering still needs verification.
  • Cleanup mismatch: GuiModel had three vertex buffers but only deleted two. This was fixed while adding instance buffers.
