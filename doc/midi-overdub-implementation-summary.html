MIDI Overdub Implementation Summary

Date: 2026-05-31

Contract

Implemented against .copilot/plan.md as the source contract.

What Was Implemented

  • Pure MIDI overdub helper surface in MidiOverdub for source-note copy, punch-window erasure, note boundary splitting, and canonical ordering.
  • Bulk install/build APIs on MidiLoop for deterministic non-RT final event installation and quantisation/model compatibility.
  • LoopTake MIDI overdub session wiring for source snapshot mapping, punch-window tracking, held-note capture, and final merge/render at Play.
  • Station trigger integration to pass source take information through existing overdub trigger flow.
  • Helper/unit/integration tests for helper behavior, loop install path, loop-take overdub behavior, and station overdub handoff.

Rubber-Duck One-Note Lifecycle Review

Reviewed one-note flow: source copy -> punch erase -> live capture -> finalize merge -> playback -> loop wrap/flush.

Issue found and fixed: zero-length punch windows could stage synthetic NoteOn/NoteOff at the same sample, and canonical ordering could leave a ghost held note. Fix: track the live-event start index at punch-in and roll back staged live events when punch-out closes a zero-length window.

Residual Risks

  • Very dense overdub event material still truncates at fixed loop capacity; this is deterministic but can drop tail content.
  • Device/channel mapping fallback behavior is intentionally permissive for legacy source takes with missing device names.
