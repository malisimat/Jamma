Bugfix Summary: Overdub Punch-In Bounce Routing

Workspace: Jamma-overdub-punchin

Implemented Tests Passing Narrow Scope Change

Bug

During overdub punch-in flow, bounced source-loop audio was being written to one destination channel only. Live ADC audio during punch-in still reached both new loops.

Observed in trigger-mediated source bounce path.

Root Cause

Source loop bounce writes were routed by trigger input-channel mapping instead of preserving each source loop's channel index. With mono trigger input mapping, both source loops were funneled into one target channel.

Code Changes

  • Added regression test: AudioFlow.TriggerBounceRoutesSourceLoopsToMatchingTargetChannels.
  • Changed trigger bounce API to accept explicit destination channel in Trigger::WriteBlock(..., destChannel).
  • Updated loop bounce call site to pass each loop's channel: Loop::WriteBlock -> trigger->WriteBlock(..., LoopChannel()).
  • Kept punch-in fade behavior by continuing to use trigger mixer level for bounce fade coefficients.

Validation

  • AudioFlow.TriggerBounceRoutesSourceLoopsToMatchingTargetChannels: PASS
  • AudioFlow.TriggerPunchActionsMuteSourceTakeAndToggleTargetState: PASS
  • *BounceProducesNonZeroAndAlignedOutput* (4 parameterized overdub cases): PASS
