Jamma MIDI loop visualization

Integration slice complete

This pass connected the previously built MIDI span extraction, instanced note model, and shader foundation into LoopTake scene ownership and update flow.

Status

Implemented

  • MIDI loops now own an attached MidiModel.
  • LoopTake creates, arranges, draws, and updates MIDI rings.
  • Record-end model rebuild produces final note spans immediately.

Realtime Boundary

MIDI event ingestion remains lightweight. RecordMidiEvent only stamps and records events; model extraction and GL-facing instance data staging happen through non-realtime update paths.

Verification

  • Focused MIDI visualization tests: 26 passed.
  • Full native suite: 308 passed, 1 skipped.
  • Debug x64 app target: build succeeded.

What Changed

  • MidiLoop gained model attachment, cached revision/length tracking, and a non-realtime UpdateModelFromEvents path.
  • LoopTake::Record now creates a MidiModel for each armed MIDI channel and adds it as a 3D child.
  • LoopTake::_ArrangeChildren now lays out audio and MIDI rings as one concentric visual set.
  • LoopTake::_UpdateLoops updates MIDI models from recorded events outside the MIDI hot path.
  • LoopTake::Play forces a final MIDI model rebuild after EndRecord, so clamped/unmatched notes are visible immediately.
  • Loop exposes its display fraction so MIDI rings can follow audio ring rotation when audio loops are present.
  • LoopTake tracks a lightweight MIDI visual playback cursor for MIDI-only takes.

Tests Added

  • MidiLoop.AttachedModelUpdatesFromRecordedNoteSpans verifies note spans reach the attached model.
  • MidiLoop.AttachedModelClampsRecordingNoteToDisplayLength verifies live recording display length clamps an open note.
  • LoopTakeMidiVisualization.RecordCreatesMidiModelChild verifies MIDI recording creates a model child.
  • LoopTakeMidiVisualization.PlayFinalizesMidiModelSpans verifies record end finalizes note-span instances through LoopTake.

Remaining Check

Manual

The app compiles, but live visual confirmation still needs a MIDI-capable manual run: record short and long notes, confirm concentric MIDI rings, hover/picker behavior, and rotation alignment in the actual scene.

Implementation note: The live update path uses MidiLoop::Revision() plus display length throttling, so repeated jobs do not rebuild unchanged MIDI geometry every block. The final record-end update is forced to catch unmatched notes clamped to the loop end.
