Jamma Channel Indexing and Routing: What the Code Actually Does

This document validates the current implementation for record and playback routing across ChannelMixer, Station, LoopTake, and Loop, with special attention to channel index transforms, trigger behavior, and practical edge cases.

Derived from source review on 2026-05-22 (Jamma-vst3 worktree)

1) Quick Verdict on Your Assumptions

Assumption Verdict What Actually Happens
ADC input 0..N-1 maps 1:1 to ChannelMixer Confirmed ChannelMixer::FromAdc writes each hardware channel index directly into per-channel ADC ring buffers.
ChannelMixer maps to Station based on Trigger wiring Incorrect ChannelMixer::WriteToSink always calls dest->OnBlockWriteChannel(chan,...) by channel index. Trigger wiring does not participate in this index map.
Station passes to LoopTake based on Trigger wiring Partial Station fans record sources (ADC/MONITOR/BOUNCE) to all takes by the same channel. Triggers decide which takes are armed/created, not per-sample channel remap.
LoopTake 0..numLoops-1 maps based on Trigger wiring to Loop Incorrect LoopTake::_InputChannel maps directly by index: channel -> _loops[channel] for record sources.
Loop is mono (single stream) Confirmed Loop reads mono sample stream, then mixer/trigger behavior distributes to destination channels.
Playback Loop to LoopTake buffer is 1:1 first loop to first buffer Conditionally true Loop output channel is seeded from LoopParams.Channel (ADC channel at creation), so routing is by loop channel tag, not guaranteed loop slot index.
LoopTake to Station buffer is matched by index Default only Default route is i -> i, but LoopTake per-buffer mixers can route any loop buffer to any Station bus channel via rack routing.
Station to DAC is routed by GuiRack/Wire behavior Confirmed Station per-bus mixers route bus buffers into ChannelMixer DAC sink channels; ChannelMixer::ToDac then copies sink channels to hardware outputs.

2) Record Path (Exact)

ADC in[ch] -> ChannelMixer ADC buffer[ch] -> Station OnBlockWriteChannel(ch) -> LoopTake OnBlockWriteChannel(ch) -> LoopTake _loops[ch]
  • Each callback writes monitor then ADC pass (different request.source values), but channel indices are unchanged.
  • Station does not remap channel numbers for record sources; it fans out by same channel index to each take.
  • LoopTake record-source mapping is strict index lookup (_loops[channel]), so loop count must cover the channel index.
Important implication: recording channels like [2,3] into a take with two loops can miss writes, because valid loop slots are [0,1] while incoming channels are [2,3].

3) Playback Path (Exact)

Loop mono read -> Loop mixer (Wire/Pan/etc.) -> LoopTake audioBuffers[*] -> LoopTake per-buffer mixers -> Station audioBuffers[*] -> Station per-bus mixers -> ChannelMixer DAC sink[*] -> Hardware out[*]
  • Loop sends with source type AUDIOSOURCE_MIXER; LoopTake maps that to _audioBuffers[channel].
  • LoopTake and Station each run a two-stage model: aggregate into intermediate buffers, then route/mix out through their own mixer arrays.
  • Both stages can run VST chains before final per-channel routing to parent sink.

4) Trigger Wiring: Where It Matters vs Where It Does Not

  • Trigger input channels define which channels are requested when creating new record/overdub takes.
  • Trigger does not remap regular ADC channel numbers during ChannelMixer -> Station -> LoopTake record fanout.
  • During overdub bounce (Station::OnBounce), Trigger participates by routing loop audio through Trigger::_overdubMixer using Bounce behavior channels.
  • So trigger wiring matters strongly for overdub control path, but not for base ADC channel-index transport.

5) Your 2-Loops to Separate Stereo Pairs Scenario

Goal: Loop A -> out 0/1 with its own pan, Loop B -> out 2/3 with its own pan.

Part Current Status Notes
Separate stereo-pair assignment Supported Set LoopTake mixer routes so buffer 0 routes to {0,1}, buffer 1 routes to {2,3}.
Independent per-loop panning (different L/R gains) Not fully supported in LoopTake stage LoopTake per-buffer mixers are Merge/Wire-style channel selection. No per-output gain vector at that stage without behavior changes.
Works with non-contiguous source channels (e.g., input 2 and 3) Broken today Code currently conflates channel index and loop slot in both record and playback handoff points.

6) Confirmed Divergences and Bugs

A separate fix plan is provided in doc/channel-routing-fix-plan.md.
  • Confirmed bug: LoopTake assumes loop slot index == channel index in record and playback handoff code paths.
  • Confirmed bug: ChannelMixer channel count shrink path does not actually shrink internal buffers due incorrect resize target.
  • Divergence from assumption: Trigger wiring is not the primary index mapper for normal record fanout; direct channel index transport is.

If these bugs are fixed, the routing model will align much better with user mental model and multichannel edge cases.
